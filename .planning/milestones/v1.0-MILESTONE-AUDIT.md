## Tech Debt Register

### MEDIUM — ERR-04 Saxon URI path parsing

**Component:** `PreviewManager.java` lines 34–47, `LogController.java` line 131
**Issue:** When Saxon throws `SaxonApiException`, `ErrorManager.extractLocation()` returns a location string with a `file://` URI as systemId (e.g., `file:///Users/dev/project/transform.xsl`). `PreviewManager.toPreviewErrors()` stores this as the `file` field of `PreviewError`. `LogController` then calls `Path.of(entry.file())` — `Path.of("file:///...")` does not produce a valid filesystem path.
**Impact:** Click-to-navigate from Saxon XSLT runtime error rows may silently fail or navigate to wrong path. FOP errors and validation errors (which use plain path strings) are unaffected.
**Fix:** In `PreviewManager.toPreviewErrors()`, add URI decoding: `String filePath = file.startsWith("file://") ? java.net.URI.create(file).getPath() : file`.

### LOW — EDIT-06 Occurrence Highlighting Partial

**Component:** `OccurrenceHighlighter.java`
**Issue:** Token stripping improved (outer XML punctuation stripped before regex search), but some edge cases remain (e.g., selecting content across element boundaries). Wired and functional for common cases.
**Fix:** Improve token normalization in a future patch.

### LOW — EDIT-07 Go-to-Definition Unconfirmed

**Component:** `HrefExtractor.java`, `EditorController.buildTab()` Ctrl+Click InputMap
**Issue:** Implementation committed (05-04), wired in EditorController. Human verification during Phase 5 execution (05-05 checkpoint) did not explicitly test Ctrl+Click navigation.
**Fix:** Human-test the Ctrl+Click flow on an XSLT file with `xsl:include` href.

### LOW — ValidationEngine null guard

**Component:** `ValidationEngine.java` line 48
**Issue:** `project.xmlInput()` used without null check. Blocked in E2E wiring by `RenderController.handleRender()` line 93. `RenderOrchestrator.render()` (public, tested directly) does not have this guard.
**Fix:** Add `if (project.xmlInput() != null)` guard before adding to files list.

### LOW — Missing VERIFICATION.md for Phases 01, 05, 07

**Issue:** Documentation gap only. All three phases have complete SUMMARY.md files with requirements evidence. Phases 02, 04, 06 have `human_needed` verifications (automated checks passed, UI tests pending human confirmation).
