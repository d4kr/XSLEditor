# Milestone v1.0: MVP

**Status:** ✅ SHIPPED 2026-04-21
**Phases:** 1–9
**Total Plans:** 24

## Overview

Full-featured local desktop developer tool for editing multi-file XSLT/XSL-FO templates, generating PDFs on demand, and debugging locally. Built in Java 21 with JavaFX. Covers the complete XML → XSLT → XSL-FO → PDF pipeline with an integrated editor, file tree, render pipeline, PDF preview, error log, and test suite.

## Milestone Summary

**Key Decisions:**

- Shadow plugin → com.gradleup.shadow 9.0.0-beta12 (com.github.johnrengelman.shadow incompatible with Gradle 9)
- WebView + PDFBox 150 DPI PNG rendering (macOS WebKit does not render PDF via file://)
- EditorTab public final fields (data carrier, no setters)
- Dirty state via UndoManager.atMarkedPositionProperty() (handles undo-to-clean correctly)
- Per-CodeArea Ctrl+S via WellBehavedFX Nodes.addInputMap (scene-level causes focus bugs)
- Consumer callback seam for log panel (decouples sub-controllers)
- Set Entrypoint/Set XML Input disabled in Phase 2, enabled by Phase 3 tree-selection bindings (D-04)

**Issues Resolved:**

- PDFViewerFX not on Maven Central → replaced with WebView + PDFBox rendering
- Platform.runLater anti-pattern in RenderController → removed, Task callbacks on FX thread via setOnSucceeded/setOnFailed
- REQUIREMENTS.md traceability table stale (all rows "Pending") → replaced at milestone close

**Technical Debt Incurred:**

- ERR-04: Saxon XSLT runtime errors return `file://` URI systemId — Path.of("file:///...") fails on macOS. Fix: URI-decode in PreviewManager.toPreviewErrors() when file starts with "file://"
- EDIT-06: Occurrence highlighting partial — some edge cases with content across element boundaries unresolved
- EDIT-07: Go-to-definition (Ctrl+Click) wired but not explicitly human-verified in Phase 5 execution
- Missing VERIFICATION.md for Phases 01, 05, 07 (documentation gap only)
- Nyquist compliance PARTIAL — not systematically applied in v1.0; address in v1.1
