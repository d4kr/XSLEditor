# Milestone v0.2.0: Developer UX Improvements

**Status:** ✅ SHIPPED 2026-04-22

## Overview

Fixed Saxon error navigation (ERR-04 carried from v0.1.0), added About dialog with version/stack/credits display, and added a ChatGPT quick-link button per error row in the log panel. All three features ship developer quality-of-life improvements without touching the core render pipeline.

**Depends on**: Phase 9 (v0.1.0 complete)

## Milestone Summary

**Key Accomplishments:**
- Fixed URI-decode bug so Saxon runtime errors with percent-encoded `file://` paths navigate correctly on macOS
- Added `PreviewManagerTest` with 3 unit tests covering the URI edge cases
- Implemented version.properties injection via Gradle processResources; static HostServices accessor for browser-open
- Built full dark-themed About dialog (programmatic, no FXML) showing app version, Java/Saxon/FOP/JavaFX runtime versions, author, and license link
- Wired Help menu → About dialog end-to-end (FXML + FXML controller)
- Added 5th column (`colAi`) to error log TableView with 💬 button per row; opens ChatGPT pre-filled with Italian-preamble prompt; event filter prevents row-click side effects

**Key Decisions:**
- `TableCell<T, Void>` pattern for action-button columns (no cell value, only graphic)
- Italian prompt preamble hardcoded per product intent (tool targets Italian-speaking devs)
- `URLEncoder.encode + replace('+','%20')` per T-12-01 threat mitigation
- `addEventFilter(MOUSE_PRESSED)` on button to block row-click propagation (WR-01 fix)
- `version.properties` injected at build time via Gradle processResources (not runtime classpath scan)

**Tech Debt Incurred:**
- EDIT-06 occurrence highlighting edge cases — still deferred
- EDIT-07 Ctrl+Click go-to-definition with xsl:include — human verification still pending
- Missing VERIFICATION.md for phases 01, 05, 07 (v0.1.0 debt)
- Phase 11 About dialog not covered by automated tests
