## Spacing Scale

JavaFX spacing maps to `-fx-padding` and `-fx-spacing` in CSS, or `setSpacing()` / `setPadding()` in code.
All values are multiples of 4.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline padding |
| sm | 8px | Menu item internal padding, toolbar button gaps |
| md | 16px | Dialog content area horizontal padding |
| lg | 24px | Dialog header bottom margin |
| xl | 32px | Minimum dialog width padding |
| 2xl | 48px | Not used in Phase 2 |
| 3xl | 64px | Not used in Phase 2 |

Exceptions:
- `DirectoryChooser` and `TextInputDialog` spacing is OS-native — not overridden.
- JavaFX `Alert` dialog minimum width: 360px (set via `alert.getDialogPane().setMinWidth(360)`).

## Interaction States

### Menu Items

Phase 2 adds the following items to the `File` menu (in order):

```
File
  Open Project...        [always enabled]
  ─────────────────────
  Set Entrypoint         [disabled when: no project loaded OR no file selected in tree]
  Set XML Input          [disabled when: no project loaded OR no file selected in tree]
  ─────────────────────
  New File...            [disabled when: no project loaded]
  ─────────────────────
  Exit
```

Disabled state: `-fx-opacity: 0.4` on the MenuItem, or use `menuItem.setDisable(true)` (JavaFX default disabled rendering is acceptable — do not override with custom CSS unless the default is illegible on dark background).

### DirectoryChooser

- Title: `"Open Project"` (set via `directoryChooser.setTitle("Open Project")`)
- Initial directory: `System.getProperty("user.home")` if no project previously loaded in this session
- OS-native dialog — no custom styling applied
- On cancel (null returned): no action, no error shown
- On confirm: load project, update window title via `updateTitle(projectName)`

### TextInputDialog — New File

- Title: `"New File"`
- Header text: `"Create a new file in the project root"`
- Content text label: `"File name:"`
- Default value: `""` (empty — force user to type)
- Prompt text (text field): `"e.g. output.xsl"`
- Validation: if the entered name is blank → show inline `Alert(ERROR)` and re-open dialog; if file already exists → show `Alert(WARNING)` with message per Copywriting Contract

### Alert — Confirmation Pattern

Inherit the pattern established in Phase 1 (`MainController.handleCloseRequest`):
- Always call `alert.initOwner(primaryStage)` — modal to the main window
- Always set explicit `ButtonType` list: no reliance on default button sets
- `showAndWait()` — never `show()`
- `AlertType.CONFIRMATION` for decisions; `AlertType.ERROR` for failures; `AlertType.WARNING` for non-blocking warnings

## Copywriting Contract

All copy in English (CLAUDE.md convention).

| Element | Copy |
|---------|------|
| Primary CTA — open project | `Open Project...` (menu item; ellipsis signals a dialog follows) |
| Primary CTA — set entrypoint | `Set Entrypoint` (no ellipsis — acts immediately on selected file) |
| Primary CTA — set XML input | `Set XML Input` (no ellipsis — acts immediately on selected file) |
| Primary CTA — new file | `New File...` (menu item; ellipsis signals a dialog follows) |
| Window title — project loaded | `XLSEditor — {projectName}` (em dash U+2014, already implemented in `updateTitle()`) |
| Window title — no project | `XLSEditor` |
| Status feedback — project loaded | `Project opened: {projectName}` (transient label or log entry, 3-second auto-hide) |
| Status feedback — config restored | `Loaded entrypoint: {filename} · XML input: {filename}` (info-level log entry) |
| Status feedback — no config found | `No .xslfo-tool.json found — entrypoint and XML input not set` (info-level log entry; not an error) |
| Empty state — no project | `No project open. Use File → Open Project... to get started.` (shown in file tree placeholder; replaces Phase 1 "File Tree" placeholder label) |
| New File dialog — title | `New File` |
| New File dialog — header | `Create a new file in the project root` |
| New File dialog — field label | `File name:` |
| New File dialog — field prompt | `e.g. output.xsl` |
| Error — blank filename | `File name cannot be empty.` (Alert ERROR, header: `Invalid File Name`) |
| Error — file already exists | `A file named "{name}" already exists in the project root. Choose a different name.` (Alert WARNING, header: `File Already Exists`) |
| Error — project load failure (IO) | `Could not open project: {directory path}. Check that the directory is readable.` (Alert ERROR, header: `Open Project Failed`) |
| Error — config parse failure | `Could not read .xslfo-tool.json: {error message}. The file may be corrupted. Entrypoint and XML input were not restored.` (Alert WARNING, header: `Configuration Warning`) |
| Destructive confirmation — none | No destructive (delete/overwrite) actions exist in Phase 2. "New File" creates only; it does not overwrite. |
