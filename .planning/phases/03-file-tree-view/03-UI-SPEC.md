# Phase 3 — UI Design Contract: File Tree View

> Visual and interaction contract for Phase 3. Generated by gsd-ui-researcher, verified by gsd-ui-checker.

---

## Design System

| Property | Value |
|----------|-------|
| Tool | none |
| Preset | not applicable |
| Component library | JavaFX 21 built-in controls |
| Icon library | Unicode glyphs via JavaFX Label (no external library) |
| Font | System Monospaced (inherited from Phase 1 `.placeholder-label`) |

**Rationale:** Project uses pure JavaFX CSS. No external icon library is in `build.gradle`. Visual distinction for entrypoint and XML input is achieved via Unicode glyph prefixes on tree cell labels, which requires zero new dependencies.

---

## Spacing Scale

All values are multiples of 4, expressed as JavaFX `-fx-padding` or `-fx-cell-size`.

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon-to-filename gap within tree cell |
| sm | 8px | Tree cell vertical padding (top + bottom each 4px) |
| md | 16px | Tree panel horizontal padding (left inset) |
| lg | 24px | Panel header height (tree panel title bar) |
| xl | 32px | Not used in this phase |
| 2xl | 48px | Not used in this phase |
| 3xl | 64px | Not used in this phase |

Exceptions: Touch targets not applicable (desktop-only tool). Tree cell minimum height is 24px to ensure clickability with mouse.

---

## Typography

| Role | Size | Weight | Line Height | JavaFX Property |
|------|------|--------|-------------|-----------------|
| Tree cell filename | 13px | Regular (400) | 1.3 | `-fx-font-size: 13px; -fx-font-family: "Monospaced"` |
| Panel header label | 12px | Regular (400) | 1.2 | `-fx-font-size: 12px` |
| Empty state message | 13px | Regular (400) | 1.4 | `-fx-font-size: 13px; -fx-font-family: "Monospaced"` |

**Exactly 2 sizes (12px, 13px) and 1 weight (regular).** No bold or larger heading is introduced in this panel — it is a supporting panel, not a focal area.

---

## Color

All values from the established Phase 1 dark palette. No new colors are introduced.

| Role | Hex | Usage |
|------|-----|-------|
| Dominant (60%) | `#2b2b2b` | Tree panel background (inherited from `.placeholder-pane`) |
| Secondary (30%) | `#1e1e1e` | Tree cell selected state background |
| Accent — entrypoint (10%) | `#66bb6a` | Entrypoint file label text color only |
| Accent — XML input (10%) | `#64b5f6` | XML input file label text color only |
| Default file text | `#cccccc` | All other files in the tree |
| Muted / placeholder text | `#888888` | Empty state label when no project is open |
| Selection highlight border | `#555555` | Tree cell focus ring border (matches SplitPane divider color) |
| Destructive | not applicable | No destructive action in this phase |

**Accent reserved for:**
- Entrypoint file row text: `#66bb6a` (same green as `status-label-success` — reuses established success tone to signal "active/primary")
- XML input file row text: `#64b5f6` (light blue — distinct from green, signals "input/data")

These two accent colors are applied ONLY to the text of the specifically designated files. No other rows, headers, or UI elements in this panel use color beyond the default `#cccccc`.

## Component Specification

### TreeView

```
Component:  javafx.scene.control.TreeView<FileItem>
fx:id:      fileTree  (replaces contents of fileTreePane StackPane)
Root node:  hidden (showRoot = false); root label = project directory name
Cell type:  Custom TreeCell subclass: FileItemTreeCell
```

### FileItem Model

```java
// Data model for tree nodes
record FileItem(Path path, FileRole role) {
    enum FileRole { ENTRYPOINT, XML_INPUT, REGULAR }
}
```

### FileItemTreeCell Visual Rules

| Condition | Glyph prefix | Text color CSS |
|-----------|-------------|----------------|
| `role == ENTRYPOINT` | `\u25B6 ` (filled right triangle, U+25B6) | `-fx-text-fill: #66bb6a` |
| `role == XML_INPUT` | `\u25A0 ` (filled square, U+25A0) | `-fx-text-fill: #64b5f6` |
| `role == REGULAR` | `\u25A1 ` (empty square, U+25A1) | `-fx-text-fill: #cccccc` |

**Glyph rationale:** Unicode geometric shapes are available in all Monospaced fonts, require no external dependency, and render crisply at 13px. Filled triangle for entrypoint (directional — "this is the start"), filled square for XML input (solid — "this is the data"), empty square for regular files (neutral).

### Interaction Contracts

| Interaction | Behaviour |
|-------------|-----------|
| Single click | Selects the row; highlights with `#1e1e1e` background; no side effect |
| Double click | Opens the file in the editor (calls `editorPane` integration point — see Phase 4 contract) |
| Right click | No context menu in Phase 3 (deferred) |
| Tree refresh | `TreeView.setRoot()` called with fresh root whenever `ProjectContext.files` changes |
| Keyboard: Enter | Same as double click — opens selected file |
| Keyboard: Arrow keys | Standard JavaFX TreeView navigation — no override needed |

### Panel Header

A `Label` sits at the top of the `fileTreePane` StackPane above the TreeView, reading the project directory name. When no project is open, the existing Phase 1 placeholder Label is shown unchanged.

```
Header label:  project root directory name (e.g. "my-templates/")
Font:          12px, #888888
Padding:       8px horizontal, 4px vertical
Background:    #1e1e1e (matches log panel title bar)
```

### Panel Layout (within fileTreePane StackPane)

```
fileTreePane (StackPane, 20% of outer SplitPane)
 └─ VBox
     ├─ Label (panel header: project dir name)  — height: 24px
     └─ TreeView<FileItem>                       — grows to fill remaining height
```

When no project is loaded: the existing placeholder Label from Phase 1 FXML is preserved. The VBox/TreeView is added programmatically when a project opens.

## Copywriting Contract

| Element | Copy |
|---------|------|
| Primary CTA | Not applicable — no primary CTA button in this panel |
| Panel header (project loaded) | `{directoryName}/` — e.g. `my-templates/` |
| Empty state heading (no project) | `No project open. Use File → Open Project... to get started.` (unchanged from Phase 1) |
| Empty state body | _(same single label — no separate body copy)_ |
| Tooltip: entrypoint glyph | `Entrypoint XSLT` |
| Tooltip: XML input glyph | `XML Input` |
| Tooltip: regular file glyph | _(no tooltip — unambiguous)_ |
| Error state (file cannot be read) | `Could not read project files. Check directory permissions.` |
| Destructive confirmation | Not applicable — no destructive action in this phase |

**Tooltip spec:** Tooltips appear on hover after 600ms (JavaFX default). Applied to the glyph Label portion of the cell only, not the full row.

## State Matrix

| App State | Tree Panel Shows |
|-----------|-----------------|
| No project loaded | Phase 1 placeholder Label (`No project open...`) |
| Project loaded, files present | TreeView with FileItem rows |
| Project loaded, empty directory | TreeView root visible, no children; no additional empty message |
| Project loaded, entrypoint set | Entrypoint row uses green text + filled triangle glyph |
| Project loaded, XML input set | XML input row uses blue text + filled square glyph |
| New file created | Tree refreshes automatically; new file appears as REGULAR |
| Entrypoint changed | Previous entrypoint row reverts to REGULAR style; new entrypoint gains green |
| XML input changed | Same pattern as entrypoint change |
