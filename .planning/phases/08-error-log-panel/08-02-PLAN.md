<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| RenderController.result.errors() → LogController.setErrors | PreviewError list crosses sub-controller boundary. Originates from Saxon/FOP parsing user's local XSLT/XML. |
| LogEntry.file (derived from PreviewError.file) → EditorController.navigateTo(Path.of(...)) | Path value used as filesystem target. |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-08-04 | Tampering | `Path.of(entry.file())` in LogController row click — malicious file string could reference paths outside the project root | accept | Local desktop tool, single-user, `PreviewError.file` is produced by Saxon/FOP parsing the user's OWN project files. Even in the worst case (malformed XSLT crafted to produce odd error file strings), opening a file outside the project root is not a security risk — the user owns the machine. No path-traversal mitigation added. |
| T-08-05 | Denial of Service | `errorsCallback.accept(result.errors())` with very large error list | accept | Error cardinality bounded by pipeline (dozens at most). `FilteredList` is O(n); acceptable. |
| T-08-06 | Information disclosure | Full Saxon error text (may quote XSLT file contents) rendered in TableView | accept | Single-user local tool — content is the user's own files by definition. |
</threat_model>
