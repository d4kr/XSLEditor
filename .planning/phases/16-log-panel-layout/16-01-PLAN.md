# Plan 16-01: Log Panel TableView Layout Fix

**Phase:** 16 — Log Panel Layout
**Plan:** 16-01
**Goal:** Set `CONSTRAINED_RESIZE_POLICY` on `logTableView` and declare `minWidth`/`maxWidth` on all five columns so the table fills 100% width, eliminates the phantom filler column, and keeps columns readable at minimum window size.

---

## Context

Root cause: no `columnResizePolicy` set → JavaFX defaults to `UNCONSTRAINED_RESIZE_POLICY`, which adds a phantom filler column and does not force total column width to match container width.

Fix: single attribute change on `<TableView>` + `minWidth`/`maxWidth` declarations on each `<TableColumn>`. No Java code changes.

**File to edit:** `src/main/resources/ch/ti/gagi/xsleditor/ui/main.fxml`

---

## Threat Model

| Risk | Mitigation |
|------|-----------|
| `CONSTRAINED_RESIZE_POLICY` with total fixed widths > container squashes columns | `colMessage` has no `maxWidth` cap — absorbs/releases width freely |
| `colAi` button clipped if column too narrow | `minWidth="40" maxWidth="40"` — fixed width, immune to resize |
| `maxWidth="1.7976931348623157E308"` triggers FXML parse error | This is `Double.MAX_VALUE` as a literal; JavaFX FXML parser accepts it — confirmed pattern used in other JavaFX projects |

### Task 2 — Apply FXML patch

Replace the existing `<TableView>` block with the corrected version:

```xml
<TableView fx:id="logTableView"
           columnResizePolicy="CONSTRAINED_RESIZE_POLICY"
           prefHeight="120" VBox.vgrow="ALWAYS">
    <columns>
        <TableColumn fx:id="colTime"    text="Time"    prefWidth="65"  minWidth="65"  maxWidth="65"  sortable="false"/>
        <TableColumn fx:id="colLevel"   text="Level"   prefWidth="60"  minWidth="60"  maxWidth="60"  sortable="false"/>
        <TableColumn fx:id="colType"    text="Type"    prefWidth="100" minWidth="80"  maxWidth="120" sortable="false"/>
        <TableColumn fx:id="colMessage" text="Message" prefWidth="400" minWidth="120" maxWidth="1.7976931348623157E308" sortable="false"/>
        <TableColumn fx:id="colAi"      text=""        prefWidth="40"  minWidth="40"  maxWidth="40"  sortable="false"/>
    </columns>
</TableView>
```

**Constraints:**
- Preserve exact indentation of surrounding FXML.
- Do NOT change `fx:id` values — `LogController.initialize()` depends on them.
- Do NOT add or remove columns.
- Do NOT touch any other part of `main.fxml`.

**Acceptance:** FXML diff matches the above exactly. No other lines changed.
