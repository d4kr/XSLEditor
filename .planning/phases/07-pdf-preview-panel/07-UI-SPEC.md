## Spacing Scale

Declared values (multiples of 4px, mapped to JavaFX `-fx-padding`):

| Token | Value | Usage |
|-------|-------|-------|
| xs | 4px | Icon gaps, tight inline spacing |
| sm | 8px | Cell padding, compact controls |
| md | 16px | Default element spacing |
| lg | 24px | Section padding |
| xl | 32px | Layout gaps |
| 2xl | 48px | Major section breaks |
| 3xl | 64px | Page-level spacing (not used in this phase) |

Exceptions:
- Outdated banner vertical padding: 6px top/bottom (between xs and sm). Declared in CONTEXT.md D-05 as `-fx-padding: 6 12`. This 6px value is intentional for the banner — it keeps the banner compact without touching the 4-grid boundary of the content below it.
- Outdated banner horizontal padding: 12px left/right (between sm and md). Same source.

Source: 8-point scale is default; banner exception pre-populated from CONTEXT.md D-05.

### Outdated Banner Node Specification

```xml

       text="Preview outdated — last render failed"
       styleClass="preview-outdated-banner"
       managed="false"
       visible="false"
       maxWidth="Infinity"
       StackPane.alignment="TOP_CENTER"/>
```

`managed="false" visible="false"` — same pattern as existing placeholder nodes (CONTEXT.md, Code Insights).

`StackPane.alignment="TOP_CENTER"` — pinned to top edge.

## State Machine

The preview pane has exactly four display states. The executor must implement all four.

| State | `previewWebView` | `previewPlaceholderLabel` | `outdatedBannerLabel` | Trigger |
|-------|-----------------|--------------------------|----------------------|---------|
| **Initial** | hidden | visible | hidden | App start, before any render |
| **Rendering** | unchanged | unchanged | unchanged | Render in progress (no change to preview) |
| **Success** | visible | hidden | hidden | `displayPdf(byte[])` called |
| **Failed — with prior PDF** | visible (last PDF) | hidden | visible | `setOutdated(true)` called, prior render exists |
| **Failed — no prior PDF** | hidden | visible | hidden | `setOutdated(true)` called, no prior render |

Source: CONTEXT.md D-10, D-11, D-12, D-13.

## Interaction Contract

| Interaction | Behavior |
|-------------|----------|
| Scroll in PDF | Native WebView PDF plugin handles scroll (mouse wheel, trackpad). No custom handler. |
| Zoom in PDF | Native WebView PDF plugin handles zoom (pinch/Ctrl+scroll). No custom handler. |
| Banner visibility | Appears immediately when `setOutdated(true)` — no animation, no dismiss button. |
| Banner dismissal | Automatically hidden when `setOutdated(false)` (next successful render). No manual dismiss. |
| First render | WebView becomes visible, placeholder hidden. Both changes happen in one JavaFX Platform.runLater call. |
| Subsequent render failure | WebView stays visible with last PDF. Banner appears. Placeholder stays hidden. |
| Failure before first render | No banner shown. Placeholder stays visible. |

Source: CONTEXT.md D-03 (scroll/zoom), D-06 (banner dismissal), D-12, D-13.
