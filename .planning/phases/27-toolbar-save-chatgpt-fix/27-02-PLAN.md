<objective>
Fix ERR-07 by changing one method call in `LogController.createAiButton()`: `addEventFilter(MOUSE_PRESSED, ...)` → `addEventHandler(MOUSE_PRESSED, ...)`. With a filter, the event is consumed before ButtonBase's internal MOUSE_PRESSED handler runs, so the button never arms and `setOnAction` never fires. With a handler, ButtonBase's internal handler runs first (arming the button), then this user handler runs and consumes the event before it bubbles to the TableView row selector. Net effect: the 💬 button now opens ChatGPT in the browser, while still suppressing row-selection on the underlying TableRow.

Purpose: One-word fix per CONTEXT.md D-01. URL format `https://chatgpt.com/?q=TEXT` is already correct (D-02) — no URL changes.

Output: Modified `LogController.java` with the single token swap; no other change.
</objective>

```java
// D-05: consume MOUSE_PRESSED before it bubbles to the TableView row handler
b.addEventFilter(javafx.scene.input.MouseEvent.MOUSE_PRESSED,
        mouseEvt -> mouseEvt.consume());
```

JavaFX event semantics:
- addEventFilter runs during the CAPTURING phase (top-down), BEFORE the target node's internal handlers.
  Consuming here prevents ButtonBase from arming → setOnAction never fires.
- addEventHandler runs during the BUBBLING phase (bottom-up), AFTER the target node's internal handlers.
  ButtonBase arms first; this handler runs after; consume() then prevents the event from bubbling
  up to the TableView/TableRow which would otherwise change selection.
