    private String loadVersion() {
        try (InputStream in = getClass().getResourceAsStream("/version.properties")) {
            if (in == null) return "unknown";
            Properties p = new Properties();
            p.load(in);
            return p.getProperty("version", "unknown");
        } catch (IOException e) {
            return "unknown";
        }
    }

    private String saxonVersion() {
        try {
            return net.sf.saxon.Version.getProductVersion();
        } catch (Exception e) {
            return "unknown";
        }
    }

    private String fopVersion() {
        try {
            return org.apache.fop.Version.getVersion();
        } catch (Exception e) {
            return "2.9";   // fallback per UI-SPEC
        }
    }

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| Classpath resource read | getResourceAsStream("/version.properties") reads a build-time artifact — no user input |
| Browser open via HostServices | showDocument() passes a hardcoded literal URL to the OS browser — no user-supplied data in the URL |
| Runtime reflection (Saxon/FOP version APIs) | net.sf.saxon.Version and org.apache.fop.Version are called via direct static reference — no Class.forName(), no dynamic class loading |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-04 | Spoofing | loadVersion() returning spoofed version | accept | version.properties is baked into the JAR; attacker would need write access to the JAR itself — outside threat model for a local dev tool |
| T-11-05 | Information Disclosure | Runtime stack versions exposed in UI | accept | This is intentional — version disclosure is the purpose of the dialog; no credentials or secrets involved |
| T-11-06 | Elevation of Privilege | showDocument() opening malicious URL | accept | URL is a hardcoded string literal "https://www.apache.org/licenses/LICENSE-2.0" — no user input, no variable interpolation; cannot be redirected |
| T-11-07 | Denial of Service | saxonVersion() / fopVersion() throwing unexpected exception | mitigate | Wrap both calls in try/catch(Exception) with fallback string; dialog always renders even if version API is unavailable |
</threat_model>
