<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| FXML action binding | onAction="#handleAbout" is resolved by JavaFX reflection at load time; only methods present on the declared fx:controller class are callable |
| HostServices.showDocument() | URL passed is a hardcoded string literal; no path from user input to this call |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-11-08 | Tampering | main.fxml onAction handler name | accept | FXML is a build-time artifact bundled in the JAR; cannot be modified at runtime by an attacker in a local dev tool context |
| T-11-09 | Spoofing | handleAbout() being invoked from unexpected context | accept | Handler is @FXML private — only the JavaFX framework can invoke it via the wired MenuItem; not part of any public API |
| T-11-10 | Denial of Service | showAndWait() blocking JavaFX Application Thread | accept | Dialog.showAndWait() is the JavaFX-sanctioned pattern for modal dialogs; it integrates with the event loop correctly and does not freeze the thread |
</threat_model>
