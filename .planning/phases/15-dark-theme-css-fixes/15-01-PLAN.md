# Plan 15-01 — Dark Theme CSS Fixes

## Goal

Append Phase 15 CSS rules to `main.css` so that:
- CodeArea shows dark background with light text
- TreeView cells show readable text in all interactive states
- TableView rows show readable text in all interactive states

All rules are appended after the existing Phase 8 block. No existing rules are modified.

---

## Threat Model

| Threat | Mitigation |
|--------|-----------|
| Modena `:selected:focused` override re-applies dark text | Explicit `:selected:focused` rules declared for both TreeView and TableView |
| `.entrypoint` / `.xml-input` colored text wins over selection text | Explicit `entrypoint:selected` and `xml-input:selected` overrides |
| `.log-error` / `.log-warn` severity colors override selection text | Explicit `selected .log-error` and `selected .log-warn` overrides |
| CodeArea `.content` sub-node not targeted | Both `.code-area` and `.code-area .content` styled |
| TableView cell gaps from cell-level background | Backgrounds set at `.table-row-cell` level, text-fill at `.table-cell` descendant |

---

## Tasks

### Task 1 — Append Phase 15 block to main.css

**File:** `src/main/resources/ch/ti/gagi/xsleditor/ui/main.css`

Append the following block verbatim after the last line of the file (after `.log-filter-btn:selected { ... }`):

```css
/* Phase 15: Dark theme CSS fixes — CodeArea, TreeView selected, TableView selected */

/* 15.1 CodeArea (RichTextFX) — dark background and visible caret/selection */
.code-area {
    -fx-background-color: #2b2b2b;
    -fx-text-fill: #cccccc;
}

.code-area .content {
    -fx-background-color: #2b2b2b;
}

.code-area .caret {
    -fx-stroke: #cccccc;
}

.code-area .selection {
    -fx-fill: rgba(9, 71, 113, 0.6);
}

/* 15.2 TreeView cells — selected and selected+focused states */
.file-tree-view .tree-cell:selected {
    -fx-background-color: #094771;
    -fx-text-fill: #ffffff;
    -fx-border-color: #555555;
    -fx-border-width: 0 0 0 2;
}

.file-tree-view .tree-cell:selected:focused {
    -fx-background-color: #094771;
    -fx-text-fill: #ffffff;
}

/* Entrypoint and xml-input colored cells must show white text when selected */
.file-tree-view .tree-cell.entrypoint:selected {
    -fx-text-fill: #ffffff;
}

.file-tree-view .tree-cell.xml-input:selected {
    -fx-text-fill: #ffffff;
}

/* 15.3 TableView rows (log panel) — row-level selection highlight */
.log-table-view .table-row-cell {
    -fx-background-color: #1e1e1e;
}

.log-table-view .table-row-cell:odd {
    -fx-background-color: #252526;
}

.log-table-view .table-row-cell:hover {
    -fx-background-color: #2d2d2d;
}

.log-table-view .table-row-cell:selected {
    -fx-background-color: #094771;
}

.log-table-view .table-row-cell:selected:focused {
    -fx-background-color: #094771;
}

.log-table-view .table-row-cell:selected .table-cell {
    -fx-text-fill: #ffffff;
}

.log-table-view .table-row-cell:selected:focused .table-cell {
    -fx-text-fill: #ffffff;
}

/* Severity colors must not override selection text */
.log-table-view .table-row-cell:selected .log-error {
    -fx-text-fill: #ffffff;
}

.log-table-view .table-row-cell:selected .log-warn {
    -fx-text-fill: #ffffff;
}

.log-table-view .table-row-cell:selected:focused .log-error {
    -fx-text-fill: #ffffff;
}

.log-table-view .table-row-cell:selected:focused .log-warn {
    -fx-text-fill: #ffffff;
}
```

**Constraint:** Do not modify any rule above the Phase 15 comment. Append only.
