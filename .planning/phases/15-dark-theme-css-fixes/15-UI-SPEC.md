## Component Color Contracts

### 1. CodeArea (RichTextFX)

**Problem:** `new CodeArea()` has no dark background set. Modena renders it with a light background by default. The `.code-area` selector alone does NOT cover the text region â€” only `.code-area .content` does.

| Selector | Property | Value | Requirement |
|----------|----------|-------|-------------|
| `.code-area` | `-fx-background-color` | `#2b2b2b` | UI-01: outer node dark |
| `.code-area .content` | `-fx-background-color` | `#2b2b2b` | UI-01: inner text region dark |
| `.code-area` | `-fx-text-fill` | `#cccccc` | UI-01: default text light |
| `.code-area .caret` | `-fx-stroke` | `#cccccc` | UI-01: caret visible on dark |
| `.code-area .selection` | `-fx-fill` | `rgba(9, 71, 113, 0.6)` | UI-01: text selection visible |

Syntax span classes (`.xml-comment`, `.xml-element`, etc.) already declared in main.css Phase 5 block. These must not be altered in this phase.

**Note:** RichTextFX uses `-fx-fill` for span text (not `-fx-text-fill`). Existing syntax classes already use `-fx-fill` correctly.

### 2. TreeView Cells

**Problem:** `.file-tree-view .tree-cell:selected` currently sets `background-color: #1e1e1e` but does NOT set `-fx-text-fill`. Modena's `:selected:focused` pseudo-class overrides text-fill to a dark color (or leaves white that clashes with the dark background choice), producing invisible text.

**State Matrix:**

| State | Background | Text Fill | Border | Requirement |
|-------|------------|-----------|--------|-------------|
| Default | `transparent` (parent is `#2b2b2b`) | `#cccccc` | none | UI-02 |
| Hover | `#333333` | `#cccccc` | none | UI-02 |
| Selected (not focused) | `#094771` | `#ffffff` | `#555555` left 2px | UI-04 |
| Selected + focused | `#094771` | `#ffffff` | `#555555` left 2px | UI-04 |

**Required selectors (must all be declared):**

| Selector | Property | Value |
|----------|----------|-------|
| `.file-tree-view .tree-cell:selected` | `-fx-background-color` | `#094771` |
| `.file-tree-view .tree-cell:selected` | `-fx-text-fill` | `#ffffff` |
| `.file-tree-view .tree-cell:selected` | `-fx-border-color` | `#555555` |
| `.file-tree-view .tree-cell:selected` | `-fx-border-width` | `0 0 0 2` |
| `.file-tree-view .tree-cell:selected:focused` | `-fx-background-color` | `#094771` |
| `.file-tree-view .tree-cell:selected:focused` | `-fx-text-fill` | `#ffffff` |

**Entrypoint and XML input cells in selected state:** The `.entrypoint` and `.xml-input` style classes set `-fx-text-fill` to a colored value. When selected, these MUST also show `#ffffff` â€” the `:selected` text-fill override must take priority. Declare the entrypoint/xml-input selected overrides explicitly:

| Selector | Property | Value |
|----------|----------|-------|
| `.file-tree-view .tree-cell.entrypoint:selected` | `-fx-text-fill` | `#ffffff` |
| `.file-tree-view .tree-cell.xml-input:selected` | `-fx-text-fill` | `#ffffff` |

### 3. TableView Rows (Log Panel)

**Problem:** `.log-table-view .table-cell` sets `background-color: transparent` and `text-fill: #cccccc` for the default state, but there is no `:selected` or `:selected:focused` override. Modena will paint selected rows with a blue background but leave text-fill subject to its own rules â€” producing dark-on-dark or invisible text on the dark surface.

**Row state matrix (applied at `.table-row-cell` level for consistent row highlighting):**

| State | Row Background | Cell Text Fill | Requirement |
|-------|----------------|----------------|-------------|
| Default (even) | `#1e1e1e` | `#cccccc` | UI-03 |
| Default (odd) | `#252526` | `#cccccc` | UI-03 |
| Hover | `#2d2d2d` | `#cccccc` | UI-03 |
| Selected (not focused) | `#094771` | `#ffffff` | UI-04 |
| Selected + focused | `#094771` | `#ffffff` | UI-04 |

**Required selectors:**

| Selector | Property | Value |
|----------|----------|-------|
| `.log-table-view .table-row-cell` | `-fx-background-color` | `#1e1e1e` |
| `.log-table-view .table-row-cell:odd` | `-fx-background-color` | `#252526` |
| `.log-table-view .table-row-cell:hover` | `-fx-background-color` | `#2d2d2d` |
| `.log-table-view .table-row-cell:selected` | `-fx-background-color` | `#094771` |
| `.log-table-view .table-row-cell:selected:focused` | `-fx-background-color` | `#094771` |
| `.log-table-view .table-row-cell:selected .table-cell` | `-fx-text-fill` | `#ffffff` |
| `.log-table-view .table-row-cell:selected:focused .table-cell` | `-fx-text-fill` | `#ffffff` |

**Severity cell colors in selected state:** The `.log-error` and `.log-warn` classes set `-fx-text-fill` on the Level column cell. When a row is selected, the Level cell must ALSO show `#ffffff` â€” severity color does not override selection text. Declare explicit overrides:

| Selector | Property | Value |
|----------|----------|-------|
| `.log-table-view .table-row-cell:selected .log-error` | `-fx-text-fill` | `#ffffff` |
| `.log-table-view .table-row-cell:selected .log-warn` | `-fx-text-fill` | `#ffffff` |
| `.log-table-view .table-row-cell:selected:focused .log-error` | `-fx-text-fill` | `#ffffff` |
| `.log-table-view .table-row-cell:selected:focused .log-warn` | `-fx-text-fill` | `#ffffff` |

**Severity colors in default (non-selected) state:** Unchanged â€” `.log-error` stays `#f44747`, `.log-warn` stays `#cca700`.

## CSS Selector Strategy (JavaFX-Specific Rules)

These rules MUST be followed by the implementor:

1. **Sub-node selectors are mandatory for CodeArea.** Do NOT style `.code-area` alone for background and text. The visible text region is `.code-area .content`. Both nodes must be styled.

2. **`:selected:focused` must be an explicit separate rule.** Modena applies its own overrides specifically for the `:selected:focused` pseudo-class combination. A rule for `:selected` alone is not enough â€” Modena will re-override it for the focused case.

3. **Row-level vs cell-level targeting for TableView.** Set background colors at `.table-row-cell` level (not `.table-cell`) so the entire row is highlighted. Override text-fill at `.table-cell` level inside the row selector. Do NOT set background at `.table-cell` level for selection â€” it will show gaps between cells.

4. **Specificity order in main.css.** All Phase 15 rules must be appended after the existing Phase 8 block so they have higher specificity order in the cascade. Do not insert rules earlier in the file.

5. **RichTextFX `.caret` and `.selection` selectors.** These are sub-nodes within the CodeArea virtual flow. `-fx-stroke` applies to the caret line; `-fx-fill` applies to the selection highlight rectangle.
