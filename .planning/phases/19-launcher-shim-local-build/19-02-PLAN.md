# Plan 19-02: Local jpackage Bundle with Icon

## Goal
Produce a native bundle (`.app` on macOS) from the fat JAR using `jpackage`, with `icon.png` visible in the macOS Dock — proving the packaging pipeline works locally before CI automates it.

## Requirements
- BUILD-04: jpackage bundle includes `icon.png` — visible in macOS Dock and Windows taskbar

## Prerequisites
- Plan 19-01 complete: fat JAR at `build/libs/XSLEditor-*.jar` with Launcher manifest

## Background
`jpackage` is bundled with JDK 14+. It wraps a fat JAR into a platform-native bundle. On macOS it produces a `.app` (or `.dmg` with `--type dmg`). The app icon is passed via `--icon <file>` — on macOS it must be `.icns`; on Windows `.ico`. We currently have `icon.png` in resources — this plan converts it to `.icns` (macOS) and documents the approach for Windows.

The icon conversion is a local step only; CI (Phase 20) will handle platform-specific icon formats per runner. This plan establishes the local invocation pattern that CI will replicate.

## Tasks

### 1. Locate icon file
```bash
find src/main/resources -name "icon*"
```
Expected: `src/main/resources/ch/ti/gagi/xsleditor/icon.png`

### 2. Convert `icon.png` → `icon.icns` (macOS only)
```bash
ICON_SRC=src/main/resources/ch/ti/gagi/xsleditor/icon.png
mkdir /tmp/xsleditor-icon.iconset
sips -z 16 16     "$ICON_SRC" --out /tmp/xsleditor-icon.iconset/icon_16x16.png
sips -z 32 32     "$ICON_SRC" --out /tmp/xsleditor-icon.iconset/icon_16x16@2x.png
sips -z 32 32     "$ICON_SRC" --out /tmp/xsleditor-icon.iconset/icon_32x32.png
sips -z 64 64     "$ICON_SRC" --out /tmp/xsleditor-icon.iconset/icon_32x32@2x.png
sips -z 128 128   "$ICON_SRC" --out /tmp/xsleditor-icon.iconset/icon_128x128.png
sips -z 256 256   "$ICON_SRC" --out /tmp/xsleditor-icon.iconset/icon_128x128@2x.png
sips -z 256 256   "$ICON_SRC" --out /tmp/xsleditor-icon.iconset/icon_256x256.png
sips -z 512 512   "$ICON_SRC" --out /tmp/xsleditor-icon.iconset/icon_256x256@2x.png
sips -z 512 512   "$ICON_SRC" --out /tmp/xsleditor-icon.iconset/icon_512x512.png
iconutil -c icns /tmp/xsleditor-icon.iconset \
  -o src/main/resources/ch/ti/gagi/xsleditor/icon.icns
```
Output: `src/main/resources/ch/ti/gagi/xsleditor/icon.icns` — CI macOS runners will reference this path.

### 3. Run jpackage locally (macOS)
```bash
JAR=$(ls build/libs/XSLEditor-*.jar | head -1)
jpackage \
  --type app-image \
  --name XSLEditor \
  --input build/libs \
  --main-jar "$(basename $JAR)" \
  --main-class ch.ti.gagi.xsleditor.Launcher \
  --icon src/main/resources/ch/ti/gagi/xsleditor/icon.icns \
  --java-options "--add-opens=javafx.graphics/com.sun.javafx.application=ALL-UNNAMED" \
  --dest /tmp/xsleditor-package
```
Expected output: `/tmp/xsleditor-package/XSLEditor.app`

### 4. Verify app icon in Dock
```bash
open /tmp/xsleditor-package/XSLEditor.app
```
- App launches (no JVM errors in Console.app)
- Dock shows the custom icon (not the generic Java coffee cup)

### 5. Commit `icon.icns` to repo
```bash
# icon.icns lives at src/main/resources/ch/ti/gagi/xsleditor/icon.icns
```
Does not need to be bundled inside the fat JAR — jpackage reads it directly from the filesystem. The path must be stable for CI to reference from the workspace checkout.

## Notes for CI (Phase 20)
- macOS runners: `--icon src/main/resources/ch/ti/gagi/xsleditor/icon.icns` + `--type dmg`
- Windows runners: convert `icon.png` → `icon.ico` using ImageMagick in the workflow; reference as `--icon icon.ico`
- The jpackage invocation above becomes the template for `.github/workflows/release.yml`
