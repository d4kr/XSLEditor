<objective>
Introduce the `EditorTab` model — the per-file state carrier that owns the RichTextFX
`CodeArea` and exposes a correct `BooleanBinding dirty` derived from
`UndoManager.atMarkedPositionProperty()`. Cover the dirty-state logic with JUnit 5 unit
tests so EDIT-02 has an automated feedback loop (Wave 0 gap in VALIDATION.md).

Purpose: Phase 4's entire dirty-indicator behaviour (tab title `*` prefix, aggregate
window-close prompt) depends on a correctly-initialised `UndoManager` baseline. Getting
the load order wrong (RESEARCH.md Pitfall 1) makes every tab open as dirty. Building and
testing this primitive in isolation — before the controller wires it into a TabPane —
keeps the risk localised.

Output:
- `EditorTab.java` — immutable-fields-only model class: `final Path path`, `final CodeArea codeArea`, `final BooleanBinding dirty`.
- `EditorTabTest.java` — three JUnit 5 tests exercising the dirty property transitions.
</objective>

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-04-01 | Tampering | `EditorTab` constructor receives a `null` `Path` | mitigate | `Objects.requireNonNull(path, "path")` — fail-fast `NullPointerException` at construction, no `NullPointerException` later when a tab tries to save. |
| T-04-02 | Tampering | `EditorTab` constructor receives a `null` `content` | mitigate | `Objects.requireNonNullElse(content, "")` — empty-string fallback. Matches "newly-created file from handleNewFile" path (empty file is legal). |
| T-04-03 | Information Disclosure | Test assertions include file paths | accept | Test uses `Path.of("dummy.xsl")` — no real filesystem path; no PII; low risk. |
| T-04-04 | Denial of Service | Very large `content` string passed to `replaceText` could allocate excessive memory | accept | Phase 4 scope covers XSLT/XSL-FO templates (RESEARCH.md security domain: typically < 1 MB). `Files.readString` in Plan 02 is the actual ingress; Plan 02 will document the size expectation. No size cap in this plan. |
